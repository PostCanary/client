## Objective

Add the foundation tokens and infrastructure that every Wave 2 component rewrite depends on: Draplin aesthetic CSS variables in `print-scale.css`, a `DEMO_VISIBLE_LAYOUTS` filter that hides the 5 unbuilt layouts from the customer-facing template browser, and explicit `logoUrl: null` signalling from the generator so PostcardFront can trigger the wordmark fallback render cleanly.

## Context

Wave 1 is the "foundation" phase of the GSD wave structure. Every Wave 2 file (PostcardFront, PostcardBack, OfferBox, CTABox, TrustBadges, OfferBadge) consumes either the new tokens or the wordmark fallback path. Wave 1 MUST land before Wave 2 starts — doing them in the wrong order means Wave 2 tasks get blocked midway when they reach for a token that doesn't exist.

This plan implements RESEARCH.md §10 (print-scale.css additions), §11 (templates.ts filter), §9 (TemplateBrowser.vue filter), and §12 (usePostcardGenerator.ts logoUrl signalling). It does NOT touch the visual components — that's Plan 03.

Key decisions grounded:
- D-02: Hide the 5 unbuilt layouts from the customer browser (code stays in place).
- D-04 / D-05 / D-06: Draplin aesthetic rules become CSS variables that every component consumes.
- D-09: Universal wordmark fallback, which is rendered in PostcardFront (Plan 03) but triggered by the generator signalling `logoUrl: null` explicitly (this plan).

export function getTemplateSetsForGoal(
  goalType: CampaignGoalType,

  const recommended = GOAL_TEMPLATE_MAP[goalType];
